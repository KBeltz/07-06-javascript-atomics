JS Atomic 02, The Q&A Session

What's the difference between alert() and console.log()?

The alert() method displays an alert box with a specified message and an OK button. It prevents the user from accessing other parts of the page until the box is closed. console.log () will take whatever is inside the parentheses and log it to the console below your code.

What's one situation where you might use an `alert()` to tell the user something?

For a warning message that needs to be displayed to the user.

What's one way that `alert()` might get obnoxious?

Overuse of alert() could become obnoxious due to the fact that the user cannot access other parts of the page until the box is closed. One is annoying enough. Two is excessive.

Would it make sense to use console.log() to tell the user things? Why or why not?

No. console.log() gives output to the console, which isn't immediately viewable by the user without going into the developer console. The console.log() statements are not something that would be seen by every user.

What's one way that console.log() might be a good tool for you to use?

Debugging

When do the console.log() statements get run? Is it before the first alert() gets displayed? After it gets displayed but before the user clicks "OK" (or equivalent)? After the user clicks to close the dialog box? A different time? How did you find out for sure?

Using the javascript console, I determined that the console.log() statements are run between the alerts. No console.log() statements were visible when the first alert box was open.
